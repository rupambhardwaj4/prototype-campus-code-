CampusCode Platform Documentation

CampusCode is a unified ecosystem designed to bridge the gap between academic theory and industry reality. It serves as a central hub where code is written, evaluated, and ranked, creating a gamified environment for continuous improvement.

The platform supports multiple distinct user types, ranging from institutional roles (Students, Faculty) to independent learners (Individual Users).

Note This documentation corresponds to version 1.0 (Prototype). Some features described may be simulated for demonstration purposes.

3. User Roles & Capabilities

Each user role has access to specific dashboards and features. Below is a detailed breakdown of what each user can do within the system.

3.1. The Student

The Student is the primary end-user of the educational features. They are associated with a specific institution.

  • Dashboard: View upcoming contests, assignments, and personal rank.
  • IDE: Write, compile, and submit code for evaluation.
  • Analytics: Track performance against peers in the same college.

3.2. The Individual User New

The Individual User represents a developer who is not bound to a specific college curriculum. This role was introduced to allow open participation in global events.

  • Global Access: Participate in open hackathons and global coding battles.
  • Public Profile: Build a portfolio visible to potential employers.
  • Restrictions: Cannot access college-specific internal exams or faculty-restricted content.

3.4. Head of Department (HOD) & Admin

HODs focus on high-level metrics: pass/fail rates, faculty performance, and curriculum adherence.

Admins manage the technical infrastructure, user account provisioning, and system-wide settings.
